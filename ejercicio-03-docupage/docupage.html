Inicializar un Repositorio

Para comenzar a usar Git en un proyecto, debes inicializar un repositorio con el siguiente comando:

git init

Este comando crea un nuevo repositorio en el directorio actual, permitiendo que Git empiece a gestionar el seguimiento de cambios.

Es una de las primeras acciones que debes hacer al iniciar un proyecto nuevo en Git.

También es útil para convertir un proyecto existente en un repositorio Git.

Recuerda que este comando solo afecta a tu directorio local, no crea una copia remota del repositorio.

Clonar un Repositorio

Si deseas obtener una copia de un repositorio existente, puedes clonar el repositorio remoto usando el siguiente comando:

git clone URL-del-repositorio

Este comando crea una copia completa de un repositorio remoto en tu máquina local, incluyendo el historial de cambios.

Una vez clonado, podrás trabajar con el repositorio como si fuera uno local.

Este comando también configura automáticamente el repositorio remoto como origen de tu repositorio local.

Comandos Principales

  • Ver el estado del repositorio:
    git status
  • Añadir archivos al área de preparación:
    git add
  • Confirmar cambios:
    git commit -m "mensaje"
  • Ver el historial de commits:
    git log
  • Deshacer cambios:
    git checkout --

Estos son algunos de los comandos más comunes que utilizarás al trabajar con Git.

Es importante familiarizarse con ellos para poder manejar tu repositorio de manera eficiente.

Git también permite realizar otras acciones más complejas, como el rebasing o cherry-pick de commits.

Trabajar con Ramas

Las ramas permiten trabajar de forma aislada en distintas características de un proyecto sin afectar el código principal.

  • Crear una nueva rama:
    git branch nombre-de-la-rama
  • Cambiar a una rama:
    git checkout nombre-de-la-rama
  • Fusionar ramas:
    git merge nombre-de-la-rama
  • Eliminar una rama:
    git branch -d nombre-de-la-rama

Cuando creas una nueva rama, Git crea una copia del proyecto en ese punto específico.

Esto permite que puedas experimentar sin riesgo de afectar el código principal.

Repositorios Remotos

Git permite trabajar con repositorios alojados en servidores remotos para compartir trabajo entre diferentes colaboradores.

  • Ver repositorios remotos:
    git remote -v
  • Enviar cambios a un repositorio remoto:
    git push origin
  • Obtener cambios del repositorio remoto:
    git pull origin

Cuando trabajas con repositorios remotos, puedes colaborar con otros desarrolladores, compartir cambios y actualizar tu copia local del repositorio.

Configurar Git

Antes de comenzar a usar Git, es necesario configurar tu nombre y correo electrónico para asociarlos con tus commits.

  • Configurar tu nombre:
    git config --global user.name "Tu Nombre"
  • Configurar tu correo:
    git config --global user.email "tu-email@example.com"
  • Ver la configuración actual:
    git config --list

Estas configuraciones son importantes ya que aseguran que tu identidad se registre correctamente en cada commit que realices.
